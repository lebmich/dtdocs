Ceci est un mode traditionnel. Il est conservé pour fournir une rétrocompatibilité avec les développements effectués dans les anciennes versions de darktable et pour permettre aux utilisateurs de continuer avec leur ancienne méthode de travail sans les forcer à utiliser le nouveau flux de travail _relatif à la scène_.

Le flux de travail _relatif à l'affichage_ met davantage l'accent sur l'exécution du traitement d'image dans la partie non-linéaire _relatif à l'affichage_ du [pipeline graphique](../../darkroom/processing-modules-and-pixelpipe/_index.md). Par défaut il utilise le module [_courbe de base_](../../module-reference/processing-modules/base-curve.md) pour mapper les images de l'espace linéaire  _relatif à la scène_ vers l'espace _relatif à l'affichage_, bien que d'autres outils de mappage des tonalités (tel que le module [_courbe des tonalités_](../../module-reference/processing-modules/tone-curve)) puissent aussi être utilisés. De nombreux modules sont déplacés plus tard dans le pipeline graphique (après ce mappage des tonalités) de telle sorte qu'ils travaillent avec des valeurs de pixel (_relatives à l'affichage_) gamma-encodées plutôt qu'avec des valeurs de pixel linéairement-encodées (_relatives à la scène_).

La plupart des étapes de base requises pour développer des images dans le cadre du flux de travail _relatif à l'affichage_ sont assez similaires à celles requises pour le flux de travail _relatif à la scène_. Les principales différences résident dans le choix des modules et dans leur ordre d'apparition dans le pipeline graphique. Pour voir la différence dans l'ordre des modules entre flux de travail _relatif à l'affichage_ et flux de travail _relatif à la scène_, veuillez vous référer à la section [ordre des modules](../../special-topics/module-order.md).

---

**Remarque:** Cette section décrit un certain nombre de modules originels qui ne sont plus recommandés pour une utilisation dans le flux de travail relatif à la scène. Il est recommandé aux nouveaux utilisateurs de se référer à la section [développement d'une image : flux de travail relatif à la scène](./edit-scene-referred.md) pour obtenir un guide sur la meilleure façon de traiter les images dans darktable.

---

balance des blancs
: Le module [_balance des blancs_](../../module-reference/processing-modules/white-balance.md) travaille de la même façon que dans le flux de travail _relatif à la scène_, par défaut, il utilise les coefficients de balance des blancs fournis par le boîtier. Si cela ne donne pas de résultats acceptables, utilisez les préréglages du boîtier ou déterminer la balance des blancs à partir d'un point neutre de votre image. Le curseur de température peut être utilisé pour rendre l'image "plus chaude" ou "plus froide". Un étalonnage des couleurs plus avancé est laissé à d'autres modules, comme indiqué plus loin.

correction exposition
: Le module [_exposition_](../../module-reference/processing-modules/exposure.md) travaille de la même façon que dans le mode _relatif à la scène_, mais la façon dont vous l'utilisez est un peu différente. Dans le mode _relatif à l'affichage_, vous devez vous assurer de ne pas trop affaiblir vos hautes lumières, et utiliser le module [_courbe de base_](../../module-reference/processing-modules/base-curve.md) pour ajuster les tons moyens si nécessaire.

: Bien que vous puissiez utiliser le module _exposition_ pour ajuster le niveau de noir pour fournir plus de contraste, vous devez faire très attention en faisant cela car vous pouvez vous retrouver avec des valeurs RVB négatives. Il est préférable d'augmenter le contraste en ajustant le pied de la _courbe de base_, cependant cela peut être un peu délicat et c'est l'une des raisons pour lesquelles le module [_filmique rvb_](../../module-reference/processing-modules/filmic-rgb.md) a été introduit dans darktable.

réduction du bruit
: Comme pour le flux de travail _relatif à la scène_, le meilleur point de départ pour la réduction du bruit est le module [_réduction du bruit (profil)_](../../module-reference/processing-modules/denoise-profiled.md).  De même, vous pouvez aussi choisir d'utiliser les modules [_réduction du bruit RAW_](../../module-reference/processing-modules/raw-denoise.mf), [_flou de surface_](../../module-reference/processing-modules/surface-blur.md), [_réduction du bruit astro photo_](../../module-reference/processing-modules/astrophoto-denoise.md), ou le module [_égaliseur de contraste_](../../module-reference/processing-modules/contrast-equalizer.md).

correction des taches
: Comme pour le flux de travail _relatif à la scène_, vous pouvez utiliser les modules [_retouche_](../../module-reference/processing-modules/retouch.md), [_correction des taches_](../../module-reference/processing-modules/spot-removal.md) et [_pixels chauds_](../../module-reference/processing-modules/hot-pixels.md) pour corriger les artefacts dans votre image.

corrections géométriques
: Comme pour le flux de travail _relatif à la scène_ vous pouvez utiliser les modules [_recadrer et pivoter_](../../module-reference/processing-modules/crop-rotate.md), [_correction de perspective_](../../module-reference/processing-modules/perspective-correction.md) et [_correction des objectifs_](../../module-reference/processing-modules/lens-correction.md) pour corriger les distorsions et recadrer votre image.

récupérer des détails
: Les images RAW contiennent souvent plus d'informations que vous ne pouvez en voir à première vue, en particulier dans les ombres. Le module [_ombres et hautes lumières_](../../module-reference/processing-modules/shadows-and-highlights.md) aide à ramener ces détails dans des valeurs de tonalité visibles.  De par la nature du capteur numérique, les détails structurels dans les hautes lumières totalement brûlées ne peuvent pas être récupérés. Cependant, vous pouvez corriger les dominantes de couleurs inopportunes dans ces zones avec le module [_reconstruire hautes lumières_](../../module-reference/processing-modules/highlight-reconstruction.md). Le module [_reconstruction des couleurs_](../../module-reference/processing-modules/color-reconstruction.md) est capable de remplir les zones surexposées avec des couleurs appropriées basées sur leur environnement. Le module [_filmique rvb_](../../module-reference/processing-modules/filmic-rgb.md) offre aussi une reconstruction des hautes lumières, mais veillez à désactiver tout d'abord le module [_courbe de base_](../../module-reference/processing-modules/base-curve.md).

ajustement des tonalités
: Presque tous les flux de travail sont susceptibles d'inclure l'ajustement de la plage des tonalités de l'image et darktable propose plusieurs modules pour aider à faire cela. Le plus basique est le module [_contraste luminosité saturation_](../../module-reference/processing-modules/contrast-brightness-saturation.md). Dans le module [_courbe des tonalités_](../../module-reference/processing-modules/tone-curve.md), les tonalités sont ajustées en construisant une courbe. Les modules [_niveaux_](../../module-reference/processing-modules/levels.md) et [_niveaux rvb_](../../module-reference/processing-modules/rgb-levels.md) proposent une interface concise, avec trois marqueurs dans un histogramme. Et bien sûr, rien ne vous empêche d'utiliser le module [_filmique rvb_](../../module-reference/processing-modules/filmic-rgb.md) dans un flux de travail _relatif à l'affichage_ si vous le souhaitez (après avoir désactivé le module [_courbe de base_](../../module-reference/processing-modules/base-curve.md)).

amélioration du contraste local
: L'amélioration du contraste local peut accentuer les détails et la clarté de votre image. Utilisé avec soin, il peut donner à votre photographie le bon piqué. Plusieurs modules sont disponibles pour cette tâche. Le module [_contraste local_](../../module-reference/processing-modules/local-contrast.md) est aisé à manipuler, avec juste quelques paramètres. Une technique beaucoup plus polyvalente, mais aussi plus complexe, est offerte par le module [_égaliseur de contraste_](../../module-reference/processing-modules/contrast-equalizer.md). Jetez un œil à ses préréglages pour avoir une idée de son fonctionnement. Le module _égaliseur de contraste_ est le "couteau suisse" de darktable pour de nombreux ajustements où la dimension spatiale joue un rôle. Notez que l'emplacement de ce module dans le pipeline graphique diffère significativement entre les flux de travail _relatif à la scène_ et _relatif à l'affichage_.

ajustements des couleurs
: darktable propose de nombreux modules pour ajuster les couleurs d'une image. Une technique très simple est implémentée dans le module [_correction des couleurs_](../../module-reference/processing-modules/color-correction.md). Utilisez ce module pour donner à l'image une teinte globale ou pour ajuster la saturation globale des couleurs. Le module [_zones de couleurs_](../../module-reference/processing-modules/color-zones.md) offre un contrôle beaucoup plus fin pour ajuster la saturation, la luminosité et la teinte dans les zones définies par l'utilisateur. Le module [_courbe des tonalités_](../../module-reference/processing-modules/tone-curve.md) -- en plus de l'ajustement classique des tonalités -- vous donne un contrôle précis des couleurs d'une image. Enfin, si vous avez l'intention de convertir une image en noir et blanc, un bon point de départ, avec une interface utilisateur simple d'utilisation et intuitive, vous est proposé par le module [_monochrome_](../../module-reference/processing-modules/monochrome.md). Vous pouvez également envisager d'utiliser le module [_calibration des couleurs_](../../module-reference/processing-modules/color-calibration.md).

netteté
: Si vous démarrez votre flux de travail à partir d'une image RAW, vous devrez augmenter la netteté de la sortie finale. Le module [_renforcer la netteté_](../../module-reference/processing-modules/sharpen.md) peut le faire avec l'approche classique USM (unsharp mask), disponible dans la plupart des logiciels de traitement d'image. Un autre moyen très polyvalent d'améliorer les bords d'une image est offert par le module [_filtre passe-haut_](../../module-reference/processing-modules/highpass.md), en combinaison avec le riche ensemble d'opérateurs de [_fusion_](../../darkroom/masking-and-blending/blend-modes.md) de darktable.

effets artistiques
: darktable est livré avec un riche ensemble de modules d'effets artistiques. Par exemple, vous pouvez utiliser le module [_filigrane_](../../module-reference/processing-modules/watermark.md) pour ajouter un filigrane à votre image. Le module [_grain_](../../module-reference/processing-modules/grain.md) simule le bruit typique des images analogiques classiques. Utilisez le module [_mappage des couleurs_](../../module-reference/processing-modules/color-mapping.md) pour transférer l'aspect et le ressenti d'une image couleur sur une autre. Le module [_faible lumière_](../../module-reference/processing-modules/lowlight-vision.md) vous permet de simuler la vision humaine pour rendre les images en faible luminosité plus proches de la réalité. Le module [_filtre dégradé_](../../module-reference/processing-modules/graduated-density.md) ajoute un dégradé neutre ou coloré à votre image pour l'exposition et la correction des couleurs.
